# ReviewTrace 하이파이 리디자인 요청서

## 앱 설명

ReviewTrace는 iPhone 화면 녹화 영상을 가져와서, 영상 속 사용자의 음성 리뷰를 타임스탬프가 있는 전사 타임라인으로 바꾸고, Codex가 바로 작업할 수 있는 리뷰 패키지로 내보내는 생산성 앱입니다.

사용자는 먼저 iPhone 기본 화면 녹화에서 마이크를 켜고 앱을 사용하면서 말로 피드백을 남깁니다. 녹화가 끝나면 ReviewTrace에서 영상을 가져오고, 앱은 영상 오디오를 전사해 시간순 타임라인을 만듭니다. 최종 결과물은 Codex 작업 패키지, 읽기용 타임라인, 원문 타임라인, JSON, SRT, VTT로 내보낼 수 있습니다.

## 핵심 사용자

초보 개발자, 1인 앱 개발자, 앱을 직접 테스트하면서 Codex에게 수정 요청을 넘기고 싶은 사람입니다. 사용자는 파일 형식과 개발 용어에 익숙하지 않을 수 있으므로, 화면은 친절해야 하지만 장황하면 안 됩니다.

## 화면별 역할

1. Home
   - 화면 녹화 가져오기를 시작하는 첫 화면입니다.
   - 사용자가 무엇을 해야 하는지 5단계로 안내합니다.
   - 최근 3개 리뷰만 최신순으로 보여줍니다.

2. Review Timeline
   - 선택한 리뷰의 핵심 결과 화면입니다.
   - 영상 길이, 전사 개수, 처리 상태를 보여줍니다.
   - 읽기용 타임라인과 원문 타임라인을 전환할 수 있습니다.
   - 타임스탬프를 누르면 영상의 같은 시간으로 이동하는 구조입니다.

3. Export
   - Codex에 보낼 기본 작업 패키지를 공유하는 화면입니다.
   - 기본 패키지는 동영상, Codex 프롬프트, 전체 전사만 포함해야 합니다.
   - 다른 형식 내보내기는 사용 목적별로 분리되어야 합니다.

4. Export Options
   - 읽기용 타임라인: 사람이 읽기 좋은 Markdown입니다.
   - 원문 타임라인: 시간 검증, 자막 기준, 원문 근거 확인용입니다.
   - JSON: 자동화와 개발자용 구조화 데이터입니다.
   - SRT/VTT: 영상 편집기나 웹 플레이어 자막용입니다.

## 디자인 방향

- 마케팅 앱처럼 화려하기보다, 신뢰감 있는 iOS 생산성 앱처럼 보여야 합니다.
- 너무 둥글고 귀여운 느낌은 피하고, 버튼과 카드 곡률은 절제해 주세요.
- 색상은 보라색을 메인 액센트로 쓰되 화면 전체가 보라색만으로 보이지 않게 중립 배경, 검정, 회색, 보조 색상을 균형 있게 써 주세요.
- 정보 구조가 명확해야 합니다. 사용자가 "Codex에 보내려면 무엇을 누르면 되는지" 즉시 알아야 합니다.
- 초보자도 이해하도록 파일 형식 설명은 짧고 자연스럽게 유지해 주세요.
- SwiftUI/iOS 네이티브 앱으로 구현 가능한 디자인이어야 합니다.

## 리디자인 요청

첨부한 ReviewTrace 스크린샷을 바탕으로 iPhone용 하이파이 UI 디자인으로 개선해 주세요.

기존 정보 구조는 유지하되 다음을 개선해 주세요.

- 홈 화면의 첫 인상과 브랜드 신뢰감
- CTA 버튼 계층: 화면 녹화 가져오기, Codex 작업 패키지 공유
- 최근 리뷰 카드의 정보 밀도와 정돈감
- 리뷰 상세 화면의 타임라인 가독성
- 읽기용 / 원문 전환 컨트롤의 명확성
- 내보내기 화면에서 "Codex용"과 "다른 형식"의 차이
- 실패/완료/처리 상태가 더 안정적으로 보이는 시각 스타일

제약:

- 랜딩 페이지처럼 만들지 마세요.
- 기능 설명 문구를 과하게 늘리지 마세요.
- 영상 분석 앱이 아니라 "앱 리뷰를 Codex 작업 문서로 바꾸는 도구"라는 정체성을 유지해 주세요.
- 한국어 UI를 기준으로 디자인해 주세요.
